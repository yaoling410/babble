# Babble Event Schema

Gemini extracts structured events from audio transcripts. This doc defines
every event type, its fields, and the full JSON shape returned by `/analyze`.

---

## Design principles

1. **Typed payloads, not generic fields.** Each event type has its own
   `details` shape — not a flat bag of nullable strings.
2. **Confidence always included.** Gemini should say how sure it is.
   Low-confidence events are shown differently in the UI (dimmed, prefaced
   with "possibly").
3. **Source quote always included.** The exact phrase from the transcript
   that produced the event. Used for debugging and for letting parents
   correct misinterpretations.
4. **Timestamps are estimates.** Audio clips are 30–90 seconds. Gemini
   should estimate event time relative to the clip timestamp, not invent
   precision that isn't there.
5. **One event = one moment.** Don't merge two feeds into one event.
   Don't split one continuous nap into two events.

---

## Event type taxonomy

```
feeding
  └── breast
  └── bottle
  └── solids
  └── pumping

sleep
  └── nap       (start or end)
  └── night     (bedtime or wake)

diaper
  └── wet
  └── dirty
  └── mixed

health
  └── fever
  └── symptom
  └── medication
  └── doctor_visit
  └── vaccine

milestone
  (domain: motor | language | cognitive | social | feeding)

mood
  (value: happy | content | alert | fussy | crying | inconsolable | drowsy)

activity
  └── tummy_time
  └── bath
  └── outing
  └── play
  └── reading
  └── music
  └── class
  └── other

growth
  (weight, height, head circumference — usually from well visits)

observation
  (anything notable that doesn't fit a structured type)
```

## Full JSON schema

### `/analyze` response

```json
{
  "new_events": [ /* Event[] */ ],
  "corrections": [ /* Correction[] */ ],
  "corrections_applied": 2,
  "summary_hint": "Two feeds, one short nap, baby seemed fussy in the morning.",
  "usage": {
    "input_tokens": 1842,
    "output_tokens": 312
  }
}
```

`summary_hint` is a one-sentence natural-language description of what happened
in this clip. Used to build the daily narrative.

---

### Event base (all events share these fields)

```json
{
  "id": "evt_abc123",
  "type": "feeding",
  "subtype": "breast",
  "timestamp": "2024-03-19T09:45:00Z",
  "timestamp_confidence": "estimated",
  "caregiver": "Mom",
  "source_quote": "nursing him right now, left side first",
  "confidence": 0.92,
  "tags": [],
  "details": { /* type-specific, see below */ }
}
```

| Field | Type | Notes |
|---|---|---|
| `id` | string | UUID, generated by backend |
| `type` | string | Top-level category (feeding, sleep, etc.) |
| `subtype` | string | More specific type within category |
| `timestamp` | ISO8601 | Best estimate of when the event occurred |
| `timestamp_confidence` | `"exact"` \| `"estimated"` \| `"unknown"` | How reliable the time is |
| `caregiver` | string? | Who was speaking / doing the action. Null if unknown. |
| `source_quote` | string | The phrase from the transcript that triggered this event |
| `confidence` | float 0–1 | Gemini's confidence. <0.6 = show as "possibly" in UI |
| `tags` | string[] | Additional labels: `first_time`, `cluster_feeding`, `new_allergen`, `urgent` |
| `details` | object | Type-specific payload (see below) |

### Feeding — breast

```json
{
  "type": "feeding",
  "subtype": "breast",
  "details": {
    "side": "left",              // "left" | "right" | "both" | null
    "duration_min": 20,          // total nursing duration, null if unknown
    "left_min": 12,              // null if not mentioned
    "right_min": 8,              // null if not mentioned
    "latch_quality": "good",     // "good" | "poor" | "refused" | null
    "issues": "unlatching repeatedly"  // free text, null if none
  }
}
```

### Sleep

```json
{
  "type": "sleep",
  "subtype": "nap",              // "nap" | "night"
  "details": {
    "phase": "end",              // "start" | "end" — log start and end as separate events
    "duration_min": 105,         // only on end events, null if start
    "location": "crib",          // "crib" | "bassinet" | "arms" | "stroller" | "car" | "contact" | null
    "position": "back",          // "back" | "side" | "front" | null — for safety flagging
    "sleep_association": "nursing",  // what got them to sleep: "nursing" | "rocking" | "pacifier" | "independent" | null
    "quality": "normal",         // "great" | "normal" | "short" | "refused" — for end events
    "night_wake": false,         // true if this is a wake during the night period
    "wake_count_tonight": null   // cumulative night wakes, on morning end event
  }
}
```

Note: start and end events are matched by the backend to calculate duration.
`sleep_start` creates a pending sleep; `sleep_end` closes it.

### Diaper

```json
{
  "type": "diaper",
  "subtype": "dirty",            // "wet" | "dirty" | "mixed"
  "details": {
    "stool_color": "yellow",     // "yellow" | "green" | "brown" | "black" | "red" | "white" | null
    "stool_consistency": "normal", // "normal" | "loose" | "watery" | "hard" | "mucousy" | null
    "blowout": false,
    "rash_noted": false,
    "rash_severity": null        // "mild" | "moderate" | "severe" | null
  }
}
```

Flag rules (applied by backend, not client):
- `white` stool → tag `urgent`, notify immediately
- `red` stool → tag `urgent`
- `green` → tag `monitor`, surface note about possible causes
- Rash `severe` → tag `urgent`

## Gemini prompt guidance

When extracting events, Gemini should:

1. **Extract all events mentioned, even in passing.** "She had a bottle earlier
   and then we went for a walk" → one bottle event + one outing event.

2. **Never invent details not in the transcript.** If amount isn't mentioned,
   `amount_oz` is `null`, not a guess.

3. **Use `confidence < 0.7` for anything ambiguous.** "She seems like she might
   be hungry" is not a feeding event — it's an observation with low confidence.

4. **Separate events that are clearly separate moments.** Two feeds mentioned
   in one clip = two events with different estimated timestamps.

5. **Tag `first_time` conservatively.** Only if the caregiver explicitly says
   "first time," "never done before," "for the first time," etc.

6. **Tag `urgent` for:**
   - Fever in infant under 3 months
   - White or red stool
   - Baby described as unresponsive or unable to wake
   - Blue/gray coloring mentioned
   - Seizure-like activity

7. **`summary_hint`** should be 1 sentence summarizing the whole clip, not
   just listing event types.
